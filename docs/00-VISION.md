# LeadPages Vision

LeadPages is a fast, practical SaaS website builder for small businesses that need leads, not complexity.

The platform should let a customer or partner create a polished, SEO-ready website quickly, then manage it without a developer.

## Primary Customers

- Tradies
- Local service businesses
- Mortgage brokers
- Agencies and referral partners
- Small businesses needing quick landing pages

## Product Promise

LeadPages should make it possible to create and manage a professional local business website in minutes.

The customer should be able to:

- Pick a trade or business type.
- Use a prebuilt theme/content pack.
- Upload photos.
- Turn sections on/off.
- Add services.
- Capture leads.
- Track enquiries.
- Connect a domain.
- Improve SEO.
- Manage the site without a web developer.

## Business Model

LeadPages supports:

- One-off website build fees.
- Monthly recurring hosting/software fees.
- Partner/reseller commissions.
- Domain registration and DNS management.
- Optional marketplace add-ons.
- Future training/support systems for partners.

## Partner Model

Partners can generate and manage customers, but LeadPages remains the core platform owner.

Important partner rules:

- Clients may see the partner as their support contact.
- LeadPages must remain able to support clients in the background.
- Client ownership should be transferable if a partner leaves or fails standards.
- Partner commissions should be measurable and auditable.
- Partner dashboards should be professional and simple.

## Product Direction

The system should become:

- More modular
- More documented
- Safer for AI coding tools
- Easier to onboard developers into
- More powerful without becoming harder for customers

## Design Direction

The interface should feel like a professional SaaS command centre.

Avoid:

- Random floating controls
- Messy admin pages
- Demo-looking UI
- Unclear groupings
- Developer-only language

Prefer:

- Clean grouped sections
- Professional navigation
- Clear labels
- Fast workflows
- Strong visual hierarchy
- Mobile-aware layouts

# Bean Culture — Superuser Pilot Guide

## Goal

Create a **new draft** event-coffee website for Bean Culture via Website Studio.  
Do **not** modify or replace the existing live Bean Culture site (`beanculture`).  
Do **not** publish the generated pilot site.

## Brief entry (UI)

Use the Website Studio brief form. Recommended field content is in:

`fixtures/website-composer/bean-culture-pilot-brief.js`

Enter real business contact details manually. Use a **designated test recipient** for leads.

Do not invent awards, prices, client logos, or testimonials. Mark unconfirmed claims.

## Expected concept shape

Plausible apps/sections:

- Event-focused hero  
- `packageCompare` — coffee cart / van / caravan  
- Event types / services  
- Gallery (`featuredProjects`)  
- Brand story / trust (only with real content)  
- Booking CTA + quote form  
- FAQ  
- Closing CTA / footer  

No plumbing, landscaping, electrical, or unrelated trade content in visible copy, SEO, JSON-LD, alt text, image queries, nav, or disabled sections.

## Application

1. Approve concept for application  
2. Confirm contacts/forms  
3. Import approved images to Cloudinary (`execute:true`)  
4. Create website from this design → **new draft site**  
5. Open editor → verify sections  
6. Preview → form test → **STOP**
